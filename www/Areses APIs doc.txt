Ghi chú: 
	+ Tất cả mọi link Web Service API đều yêu cầu phải hoàn tất authen trước khi sử dụng.
	+ Khi request phải gửi kèm theo cookie jawnSUID với nội dung là chuỗi token để kiểm tra authorise.
	+ Tất cả Response được trả về có format của JSON và với cấu trúc chung như sau:
		{
			meta://Chứa trạng thái request có thành công hay không.
			{
				success: true || false || null, //Nếu là true thì request được server thực hiện thành công, ngược lại sẽ không tồn tại hoặc nếu có sẽ mang giá trụ fasle hay null.
				error: //Nếu có lỗi phát sinh thì thuộc tính này sẽ tồn tại và có nội dung như sau:
				{
					name:"" //Tên lỗi hay mã lỗi
					message:"" //Nội dung chi tiết lỗi
				}
			},
			data: {} //Tùy vào request có thành công và có trả về dữ liệu hay không mà thuộc tính này tồn tại. Và cấu trúc cũng tùy vào dữ liệu trả về.
		}
	+ Mọi domain khác đểu có thể sử dụng mà không sợ CORS
	+ Khi nói đến user đang đăng nhập hiện tại có nghĩa là user đã authen có session gắn với jawnSUID token được gửi lên cho server thông qua cookie mỗi lần request.

1/
	URL: /api/profile/modify
	Nhiệm vụ: Thay đổi thông tin cá nhân của user đang đăng nhập hiện tại.
	Phương thức: POST
	Tham số request:
		_ Tùy chọn:
			+ eml: email
			+ pwd: mật khẩu
			+ gdr: giới tính
			+ nam: họ và tên
	Cấu trúc response:
		{
			meta:{} //Tương tự như trên
		}
